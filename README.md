## Overview

**AGPARS** ( Parsing Agent & Rental System) aggregates rental listings from 6 major Irish property websites, normalizes the data, detects duplicates across sources, and delivers real-time notifications to users via a Telegram bot.

### Supported Data Sources

| # | Website | Method |
|---|---------|--------|
| 1 | Daft.ie | Playwright + Undetected ChromeDriver |
| 2 | Rent.ie | Playwright |
| 3 | Property.ie | Playwright |
| 4 | MyHome.ie | Playwright |
| 5 | DNG.ie | Playwright |
| 6 | SherryFitz.ie | Playwright |

## Architecture

```
┌─────────────┐     ┌────────────┐     ┌────────────┐     ┌───────────┐    ┌───────────────┐
│  Scheduler   │───▶│  Collector │───▶│ Normalizer │───▶│ Publisher │───▶│   Notifier    │
│  (cron-like) │    │  (scraper) │     │  (ETL)     │     │ (outbox)  │    │  (delivery)   │
└─────────────┘     └────────────┘     └────────────┘     └───────────┘    └───────────────┘
                                                                              │
                         ┌───────────┐    ┌───────────┐                       ▼
                         │   Dedup   │    │   Rules   │              ┌───────────────┐
                         │ (linker)  │    │(exclusion)│              │ Telegram Bot  │
                         └───────────┘    └───────────┘              │ (user-facing) │
                                                                     └───────────────┘
```

### Pipeline Flow

1. **Scheduler** enqueues scrape jobs into Redis on a configurable schedule
2. **Collector** picks up jobs, scrapes listings via headless browsers (Playwright / UC driver for Cloudflare-protected sites)
3. **Normalizer** transforms raw HTML snapshots into a unified listing format, extracts price, bedrooms, location, etc.
4. **Dedup** links duplicate listings across different sources using fuzzy matching
5. **Publisher** syncs normalized listings to the `pub` schema and emits change events to an outbox
6. **Notifier** consumes outbox events, applies user subscription filters, and delivers Telegram notifications
7. **Bot** provides the user interface: subscription management, listing browsing, favorites, digest scheduling

### Database Schema

PostgreSQL is organized into 5 schemas:

| Schema | Purpose |
|--------|---------|
| `raw` | Raw scraped HTML snapshots |
| `core` | Normalized listings, dedup links |
| `ops` | Job logs, circuit breakers, event outbox |
| `pub` | Materialized views for the bot |
| `bot` | Telegram workspaces, users, subscriptions |

## Getting Started

### Prerequisites

- Docker & Docker Compose
- A Telegram bot token (from [@BotFather](https://t.me/BotFather))

### Setup

1. **Clone the repository**
   ```bash
   git clone git@github.com:Oikawhy/ireland-rental-agregator.git
   cd ireland-rental-agregator
   ```

2. **Configure environment**
   ```bash
   cp .env.prod.example .env
   # Edit .env with your credentials:
   #   POSTGRES_PASSWORD=<strong password>
   #   TELEGRAM_BOT_TOKEN=<your bot token>
   #   GF_SECURITY_ADMIN_PASSWORD=<grafana password>
   ```

3. **Start all services**
   ```bash
   cd infra/docker
   docker compose --env-file ../../.env up -d
   ```
3.1 **Start cookies scripts**   
   ```bash
   cd scripts
   python daft_cookie_login.py
   python property_cookie_login.py
   ```
4. **Verify**
   - Grafana: [http://localhost:3000](http://localhost:3000)
   - Prometheus: [http://localhost:9090](http://localhost:9090)
   - Find your bot on Telegram and send `/start`

### Running Individual Services

```bash
# Start only the bot and notifier
docker compose up -d bot notifier

# Follow logs for a specific service
docker compose logs -f collector

# Stop everything
docker compose down
```

## Telegram Bot Features

- 🏘️ **Browse listings** — filter by county, price range, bedrooms
- 🔔 **Subscriptions** — get notified when new listings match your criteria
- ⭐ **Favorites** — save listings for later
- 👁️ **Hide** — hide listings you're not interested in
- 📊 **Stats** — view scraping and delivery statistics
- ⏰ **Digest mode** — receive grouped notifications (instant / daily / paused)
- 👥 **Workspaces** — shared settings for groups / families

## Auto-Exclusion Rules

The system automatically filters out:
- ❌ Student accommodations
- ❌ Short-term rentals (< 2 months)
- ❌ Northern Ireland listings

## Monitoring

The stack includes pre-configured Prometheus metrics and Grafana dashboards for:
- Scrape job success rates and durations
- Listings found per source per cycle
- Outbox queue depth and delivery rates
- Circuit breaker states per source
- Telegram delivery success / failure rates
